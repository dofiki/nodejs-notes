Node.js Request and Response Objects

Request (req) Object:

The req object represents the incoming HTTP request made by the client to the server. It contains information about the client's request and allows the server to access various aspects of the request.

  1. URL (req.url):

    Represents the URL path requested by the client. Allows the server to determine which resource or route is being accessed.

  2. Method (req.method):

    Indicates the HTTP method used by the client (e.g., GET, POST, PUT, DELETE). Enables the server to handle different types of requests.

  3. Headers (req.headers):

    Provides an object containing all the HTTP headers sent by the client. Headers include information such as content type, user agent, and more.

  4. Body (req.body):

    Contains the data sent by the client in the request body (for POST and PUT requests). Parsing and handling the request body are common tasks when dealing with form submissions or API requests.

Response (res) Object:

The res object represents the server's response to the client's request. It allows the server to send back information, including headers and the response body.

  1. Set Header (res.setHeader(name, value)):

    Used to set HTTP headers in the response. Headers define metadata about the response, such as content type, encoding, or caching instructions.

  2. Write to Response Body (res.write(data)):

    Adds data to the response body. The response body contains the content that will be sent to the client.

  3. End Response (res.end()):

    Finalizes the response and sends it to the client. Once res.end() is called, the response is complete, and no further modifications can be made.
